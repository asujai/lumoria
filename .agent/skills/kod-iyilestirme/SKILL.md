# Code Improvement Skill

When improving code, follow these steps:

## Improvement checklist

1. **Readability**: 
   - Are variable and function names clear?
   - Is the structure easy to follow?

2. **Maintainability**:
   - Is the code modular?
   - Are responsibilities clearly separated?
   - Is duplication removed?

3. **Efficiency**:
   - Are there unnecessary loops or repeated calculations?
   - Can built-in methods improve performance?

4. **Clean Code Principles**:
   - Single responsibility principle
   - Avoid magic numbers
   - Remove dead code
   - Simplify complex conditions

5. **Best Practices**:
   - Proper error handling
   - Consistent formatting
   - Language/framework conventions

## How to provide output

- Show improved version of the code
- Explain what was changed
- Explain why it improves the code
- If multiple improvements exist, group them logically
